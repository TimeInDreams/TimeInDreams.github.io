#密钥交换协议

##要求

  1. 只有参与者知道
  2. 确保对方知道
  3. 新生成的，时效性
  • 密钥建立：密钥建立是这样一个过程，通过该过程，使得两个或多个主体拥有良好的共享密 钥
  • 会话密钥：在安全协议成功运行完毕后，主体 间进行应用层会话时使用的密钥，使用时间较 短。
  • 长期密钥：与会话密钥相对应，可以长时间多 次使用，可能是共享的对称密钥，也可能是公 开密钥系统中的公钥或私钥。

主体

密钥建立，会话密钥（短期），长期密钥

安全性质

  1. 秘密性
  2. 数据完整性：未被篡改
  3. 数据起源认证性：保证起源
  4. 非否认性：数字签名

基本目标

  • 隐式密钥认证：确信只有身份确定的参与另一方才能知道共享密钥
  • 密钥确认：参与的一方要确认另一方已经拥有了共享密钥
  • 显式密钥认证：隐式密钥认证+密钥确认

包含协议

  • 密钥传输协议：一个参与者生成密钥，分享给所有参与者
  • 密钥协商协议：所有协议使用者的生成密钥
  • 混合协议：1<n<all

简单的密钥传输协议

###前提

A,B与服务器S已建立密钥Kas,Kbs

  • 安全假设
  1. 攻击者可以窃听所有信息→秘密性，长期密钥
  2. 攻击者可以修改传输的消息
  3. 攻击者的内部属性，合法和非法→用户的名字绑定会话密钥
  4. 攻击者在一定时间以后有能力得到以前运行的协议当中的会话密钥，重放攻击

加密–中间人攻击

时效性

  • 时间戳
  • 询问-应答，随机数作为询问，返回消息也带有这一随机数

##攻击方式

窃听

篡改：数据完整性

重放

非否认性：数字签名

类型攻击：类型攻击就是利用用户接收到的都是二进制串，无法判断是否加密，使得用户将一个消息错误的解释成其他的消息→每次改变 消息元素的顺序，确保每次的加密密钥不同。 或者在消息中添加认证数或消息域的类型信息

##端到端协议

签名的本质是发送私钥的信息但不能被还原，即单向性

协议参与方的数字签名保证消息来源真实性，抵抗中间人攻击

秘密分享

具有重要价值 物件的访问权限不能只由一个人掌握。

基本思想是==将秘密分解为多个碎片==并将这些碎片分发给不同的人掌管，在秘密丢失的情况下，这些人中的某些特定子集可以通过将他们拥有的碎片凑在一起以==恢复整个秘密==

过程

  1. 秘密分发协议：分发者D，在n个参与者中分享秘密s，每个参与者Pi获得一个碎片si
  2. 秘密重构协议：任意不少于t个参与者合作，以自己碎片si为输入，重构秘密s

性质

  1. 可恢复：任意t个能回复
  2. 任意少于t个无法计算任何秘密s相关信息

经典协议

  1. 拉格朗日多项式

秘密握手

一个群组中的两个成员互相认证，同时两个成员不会互相透露他们的上下级信息，除非他们同属于一组

##流程

  1. Setup：输入成员私钥，输出公钥
  2. CreateGroup：群组授权者调用算法创建群组
  3. AddUser：GA生成证书，U获得证书通过一个被授权的私有频道
  4. HandShake：成员AB调用的一个协议，确保B发现A、当且仅当A发现B在群组中。
  5. RevoleUser：调用算法，移除成员证书

#高级签名协议

密钥生成，签名，验证

盲签名

持有消息M的用户U获取签名人Singer对M的签名s，但Singer不知道m的内容，（m，s）对公开，Singer也无法追踪消息与签名过程之间的关系

###过程

  1. Setup，输出系统参数p和签名人的公钥私钥对（pk，sk）
  2. Sign
    1. 公共输入：系统参数p，签名人公钥pk，
    2. 秘密输入，签名人私钥sk，用户输入代签名消息m
    3. 停止时用户输出签名s
  3. Verify 输入系统公开参数p，公钥pk，待验证消息签名对（m，s），输出1或0（是否有效）

性质

  1. 正确性
  2. 不可伪造性
  3. 盲性

盲化

  1. 盲化：用户引入盲化因子，将消息m计算为m` k^e^m

  2. 签名：发送m‘给签名人，签名人签名得到s’，d次幂

  3. 去盲：用户用s‘计算出s k的逆元

验证 签名s^e^(modn)=m

离散对数

(g^x^,x)为公钥私钥,用公钥+随机数去盲，用随机数盲化

  1. 选随机数r,m’=g^r^×H（m）

  2. 私钥次方s’=m’^x^

  3. s=s’y^-r^

部分盲签名，嵌入一段公共信息

群签名

成员代表群体进行匿名签名，无法确定具体成员，但群管理者可以打开签名揭示签名者身份防止否认签名行为

过程

  1. 系统初始化
  2. 签名
  3. 验证
  4. 打开

性质

  1. 匿名性：无法确定给定签名的实际签名人
  2. 无关联性：不打开签名的情况下，不能确定两个不同的群签名是否为同一个人所签名
  3. 可追踪性：
  4. 防陷害性：不能以其他群成员名义产生合法签名
  5. 抗联合攻击：不能任意多个群成员勾结伪造其他群成员签名

环签名

签名的匿名性

自己的私钥和其他n-1个人的公钥对m执行环签名，执行验证算法可以确信又环中某个签名者产生，但无法识别签名人的身份

可链接性：如果是同一个人签的，验证时能确定时同一个人签的，但无法确定身份

基于身份的数字签名

过程

  1. Setup 输出主密钥，系统参数
  2. Extract，私钥提取算法，输入为用户身份信息ID，算出私钥sk发送给用户
  3. Sign 用户签名
  4. Verify 验证者完成验证

安全性

  1. 适应性选择消息攻击
  2. 选择身份攻击的不可伪造性

类型

双线性对，RSA
