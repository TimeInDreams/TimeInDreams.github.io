总线

概念

分时共享

优点

  1. 成本低，简化硬件设计
  2. 系统扩展性高
  3. 便于维护

###缺点

总线带宽限制IO

分类

单向/双向总线

片内/系统/通信

并行/串行

###总线事务

总线上一对设备之间的一次信息交换过程

地址阶段+数据阶段

###性能指标

总线周期：一次完整数据传输的时间，总线时钟频率的倒数

主设备：发出总线事务请求

从设备：与主设备信息交互

总线带宽=一个总线周期传送的数据量/时钟周期

猝发传输：一种总线传输方式，在一个总线周期中可以传输储存地址连续的多个数据

###总线的定时控制

同步定时方式

异步定时方式：应答方式

###==仲裁==

中央仲裁器，两条控制线，总线请求信号线和总线授权信号线

  • 集中式仲裁
    • 链式查询
      • 三根控制线
        • 总线请求
        • 总线忙
        • 总线批准
      • 查询优先级固定，对查询链故障敏感（i故障，>i全部故障）
    • 计数器定时查询
      • 2（请求+忙）+log2n条控制线
      • 查询计数器计数值与发出请求的编号一致终止查询，该设备获得总线控制权
    • 独立请求
      • 2n（n请求+n授权）+1(总线忙)
  • 分布式仲裁
    • 总线请求+总线忙+数据线+n个仲裁电路

单总线，双总线，三总线，多总线

系统总线，CPU主存，IO总线

#中央处理器

##功能和组成

数据流，指令流

寄存器

专用寄存器：

  1. 程序计数器PC（将要执行的）
    1. 指令的顺序执行PC+1
    2. 转移指令目标地址
  2. 指令寄存器IR（正在执行的）
    1. 执行指令时保证指令不变
  3. 储存器地址寄存器MAR（将要访问的主存地址）
    1. 当前CPU访问的主存单元地址，CPU速度快于主存
  4. 储存器数据寄存器MDR
    1. 暂存主存写入或读出的指令或数据字
  5. 状态标志寄存器PSWR

MDR和MAR在CPU与主存进行信息交换是使用

参数

  1. 字长
  2. 内频=外频*倍频

控制器组成和实现

时序系统与控制方式

###时序系统

  1. 指令周期：取指令，分析到执行完成，每个指令的指令周期不同
  2. 机器周期：一条指令划分为若干个机器周期，有且有一个周期状态触发器置为1
  3. 节拍：CPU完成一次操作，一个机器周期划分为若干个节拍信号

指令周期–机器周期–节拍（时钟周期）–脉冲信号

指令的执行

  1. 取指令
    1. PC→MAR+AB（地址总线）
    2. 控制单元经控制总线发出read命令
    3. 通过数据总线送的MDR
    4. MDR→IR
    5. PC+1
  2. 取数周期
  3. 执行周期

微程序控制原理

机器指令对应一段程序，该段程序放在一个只读的控制寄存器中（一个指令–一个微程序）

机器指令可分解为多个微操作序列

微命令对应唯一微操作（计算机的原子操作）

微指令是控制寄存器中的一个单元的内容，是若干个微命令的集合

微地址：控制寄存器的单元地址

微指令

  1. 操作控制字段

产生若干个微操作控制信号

  1. 顺序控制字段

产生下一个微指令地址

字段编码法

将操作控制字段分为若干小段

段内最短编码（一码一指令），需要译码器

段段之间直接控制，一个段单位对应一个指令

==般每个小段还要留出一个状态，表 示本字段不发出任何微命令==

题目

指令运行速度正比于时钟频率反比于时钟周期，MIPS，每秒一百万次

如果2条微指令放在两个位足够，就是直接，操作控制字段位数不够需要3条或以上的互斥指令放在2个位上，就是段，需要注意00表示无状态

原码—补码—变补

变补

[Y]补：Y的补码，

​ 一个负数由原码表示转换成补码表示时，符号位是不变的，仅对==原码==数值位的各位变反，末尾加 “1” 。

[-Y]补：Y的补码的变补，

​ 补码表示的变补则不论这个数的真值是正是负，一律对==补码==连同符号位一起变反，末尾加 “1” 。

符号扩展

8位数 与另外一个32位数相加，将8位数扩展到32位

  1. 原有符号 位保持不变
  2. 正数则所有附加位都用0进 行填充
  3. 负数则所有附加位都用1进行填 充
  4. ==用符号位来填充附加的 高位==

补码的溢出判断与检测方法

字长为n+1位的定点整数（其中一位 为符号位），采用补码表示，当运算结果 大于2 n -1或小于-2 n时，就产生溢出。

若X、Y异号，不会溢出

带符号数的移位和舍入操作

原码负数：移位后的空出位补0

补码正数：左移右移空出位都补0

补码负数：左移后（右边空位）的空出位补0，右移（左边位）后的空出位补1。

关于数的移位，特别需要注意正数，三码相同，所以无论左移还是右移都是补0.而负数的补码就需要注意，左移在右边补0，右移需要在左边补1，有一个很有趣的误区是，认为符号位保持不变，仅仅移动数据位，这是不对的，因为无论数据位还是符号位，都是二进制，在整体大迁移的过程中，符号位也是要跟随潮流的。只不过，为了保证右移后，和原来的符号数一样，因此，负数在右移时左边补1.
