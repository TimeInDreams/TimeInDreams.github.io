原码—补码—变补

变补

[Y]补：Y的补码，

​ 一个负数由原码表示转换成补码表示时，符号位是不变的，仅对==原码==数值位的各位变反，末尾加 “1” 。

[-Y]补：Y的补码的变补，

​ 补码表示的变补则不论这个数的真值是正是负，一律对==补码==连同符号位一起变反，末尾加 “1” 。

符号扩展

8位数 与另外一个32位数相加，将8位数扩展到32位

  1. 原有符号 位保持不变
  2. 正数则所有附加位都用0进 行填充
  3. 负数则所有附加位都用1进行填 充
  4. ==用符号位来填充附加的 高位==

补码的溢出判断与检测方法

字长为n+1位的定点整数（其中一位 为符号位），采用补码表示，当运算结果 大于2 n -1或小于-2 n时，就产生溢出。

若X、Y异号，不会溢出

带符号数的移位和舍入操作

原码负数：移位后的空出位补0

补码正数：左移右移空出位都补0

补码负数：左移后（右边空位）的空出位补0，右移（左边位）后的空出位补1。

关于数的移位，特别需要注意正数，三码相同，所以无论左移还是右移都是补0.而负数的补码就需要注意，左移在右边补0，右移需要在左边补1，有一个很有趣的误区是，认为符号位保持不变，仅仅移动数据位，这是不对的，因为无论数据位还是符号位，都是二进制，在整体大迁移的过程中，符号位也是要跟随潮流的。只不过，为了保证右移后，和原来的符号数一样，因此，负数在右移时左边补1.

定点乘法运算

原码一位乘法的规则为：手算法，累加右移，符号位同号为正异号为负

X*Y需要3个寄存器：A部分积与最后乘积的高位部分初值为0，B被乘数X，C乘数Y（运算过程中不再需要保留乘数，改为存放乘积的低位部 分）

比较法－Booth乘法

  1. 补码一位乘法

  2. 符号位参加运算

  3. 乘数最低位后面增加一位附加位 Yn+1，其初值为0

  4. 每求一次部分积要右移一位， 所以乘数的最低两位Yn、Yn+1的值决定了 每次应执行的操作

  5. 移位按补码右移规则进行

  6. 共需做n+1次累加，n次移位，第 n+1次不移位

判断位 Yn Yn+1

​ 0 0 原部分积右移一位

​ 0 1 原部分积加[X]补后右移一位

​ 1 0 原部分积加[-X]补后右移一位

​ 1 1 原部分积右移一位
